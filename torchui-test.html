Notification Functions

$.notify(STRING)
Puts a notification onto the page using all default values.
$.notify(STRING, { NAME:VALUE , ...})

Creates a notification using optional parameters

Notification Options

theme
"message" : Sets the notification to the theme of a message type. This will display an evelope next to the message passed.
"loading" : This creates a loading notification into the top corner.
"debug" : This will set the message to be only shown while in debugging mode
"success" : This is to show a success message
"error" : This is to show an error message
sticky
true | false : This will make the notification 'sticky' meaning it will not dissapear on it's own, it must be clsoed
life
MilliSeconds : This is how many milliseconds the notification will live (ex : 3000 = 3 seconds)
header
STRING : This is an optional Header. It will display above the message and bold
beforeOpen
FUNCTION : function(e,m,o) {} : This is a function that wil run before the notification is opened.
open
FUNCTION : function(e,m,o) {} : This is a function that wil run when the notification is opened.
beforeClose
FUNCTION : function(e,m,o) {} : This is a function that wil run before the notification is closed.
close
FUNCTION : function(e,m,o) {} : This is a function that wil run when the notification is closed.

Ajax Options

global
true | false : this option is by default true, but you can turn off the global functions by setting it to false. **WARNING** Much functionality is handled globally by TorchUI, turn this off at your own risk **WARNING**
timeout
MilliSeconds : Default is 2500 ms so that the Ajax call will timeout after 2.5 seconds of no response.
cache
true | false : Defaults to true. This tells the browser whether to cache the information or not.
url
String (Required) : this points to a fully qualified path to a resource.
dataType
* "xml": Returns a XML document that can be processed via jQuery.
* "html": Returns HTML as plain text; included script tags are evaluated when inserted in the DOM.
* "script": Evaluates the response as JavaScript and returns it as plain text. Disables caching unless option "cache" is used. Note: This will turn POSTs into GETs for remote-domain requests.
* "json": Evaluates the response as JSON and returns a JavaScript Object.
* "jsonp": Loads in a JSON block using JSONP. Will add an extra "?callback=?" to the end of your URL to specify the callback.
* "text": A plain text string.
Default is set to "json".
data
Data to be sent to the server. It is converted to a query string, if not already a string. It's appended to the url for GET-requests. See processData option to prevent this automatic processing. Object must be Key/Value pairs. If value is an Array, jQuery serializes multiple values with same key i.e. {foo:["bar1", "bar2"]} becomes '&foo=bar1&foo=bar2'.
group
String : Assigning a group will then allow buffering of the groups. Meaining that the ajax calls are called one after another, after the previous call completes.
initialWait
MilliSeconds : Default is 800. This is used in the grouping of the ajax calls. The time will determine the length that is used to wait between the calls so they are properly throttled.

Ajax Return Functions

success
this is the function that will be called if the ajax is returned successful
error
this is the function that will be called if the ajax is returned as an error
complete
this is the function that will be called when the success or error is completed

Ajax Functions

$.ajax()
Sends an ajax call using all of the default options. This function will fail.
$.ajax({options})
The options must have {url: "String"} at the very least.
Example: $.ajax({ url: "access/some/account.do", timeout:5000, success : function(data){ doThis(data); } });

Ajax Interfaces

All ajax calls that are using the "json" data type will be subject to the interface requirements explained below.
Response
All responses will be an array.
The array will have 1 or more objects.
Each object will contain "type" and "value". Optionally a "callback" can be used.
Type defaults to 1 = success and 0 = false. Optionally you can specify others like "message" or "system failure"
Example Response from server: [ { type: 1, value : "You have successfully added a user" } ]
The response is held in an array so that multiple responses can be sent through single call if necessary.
